コマンドライン ユーザーマニュアル
単体版のユーザーマニュアルはこちらからご利用いただけます

はじめに
DeepSkyStackerのコマンドライン版は、すべてのバッチ処理を簡素化することを目的としています。
DeepSkyStackerのコマンドライン版は、DeepSkyStackerの登録（位置合わせ）およびスタッキングエンジンを共有しています。言い換えれば、コマンドライン版と単体版のどちらを使用しても、まったく同じ結果が得られます。

現在、すべての設定は、単体版のDeepSkyStackerとコマンドライン版のDeepSkyStackerの間で共有されています。これには、RAW設定、登録およびスタッキング設定、一時ファイルフォルダなどが含まれます。

つまり、コマンドライン版を使用する前に、単体版を使用して設定に合わせてすべての値を設定しておく必要があります。
今後のバージョンでは、追加のコマンドラインパラメータを使用して、一部の設定を上書きできるようになる予定です。

コマンドラインパラメータ
DeepSkyStackerのコマンドライン版は、インストール時にDeepSkyStackerフォルダに配置されるDeepSkyStackerCL.exeを使用して呼び出されます。

コマンドラインは以下のようになります。

DeepSkyStackerCL.exe [/r|R] [/S] [/SC] [/SR] [/O:<OutputFileName>] [/OFxxx] [/OCx] <ListFileName>

パラメータは以下の通りです。

/r
 		 ライトフレームを登録（すでに登録されていない場合のみ）
 
/R
 		 ライトフレームを登録（すでに登録されている場合でも - 強制登録オプションと同等）
 
/S
 		 ライトフレームをスタック
 
/SC ライトフレームをスタックし、ライトフレームごとにキャリブレーション済みのライトフレームを含むファイルを作成します。
 
/SR
 		 ライトフレームをスタックし、ライトフレームごとに登録およびキャリブレーション済みのライトフレームを含むファイルを作成します。
 
/O:<OutputFileName>



 

 出力ファイル名のオプション指定（/S または /SC を意味します）
デフォルトでは、出力ファイル名は Autosave.tif であり、最初のライトフレームフォルダに保存されます。
拡張子が別のものであっても、出力ファイルは常にTIFFファイルになります。
ファイルが作成されるフォルダが存在している必要があります。
 
/OFxxx



 		 出力ファイル形式。
指定可能な値:
/OF16 または /OF16i : 16ビット整数精度
/OF32 または /OF32i : 32ビット整数精度
/OF32r : 32ビット有理数精度（これがデフォルト値です）
/OCx



 		 出力圧縮。
指定可能な値:
/OC0 : 圧縮なし（これがデフォルト値です）
/OC1 : LZW圧縮
/OC2 : ZIP (deflate) 圧縮
/FITS 出力ファイルをFITS形式で保存します（デフォルトはTIFF）
 
<ListFileName>

 単体版のDeepSkyStackerで作成されるような、リストファイルへのフルパス
（詳細なフォーマットは下記参照）

使用例
DeepSkyStackerCL.exe /R c:\MyDSSLists\MyList.txt
ファイルリスト c:\MyDSSLists\MyList.txt 内のすべてのライトフレームを（すでに登録済みであっても）登録します。
ファイルリストにオフセット/バイアス、ダーク、フラットフレームが含まれている場合、それらは登録処理で使用され（マスターが作成され）ます。

DeepSkyStackerCL.exe /R /S c:\MyDSSLists\MyList.txt
ファイルリスト c:\MyDSSLists\MyList.txt 内のすべてのライトフレームを（すでに登録済みであっても）登録し、その後スタックします。
ファイルリストにオフセット/バイアス、ダーク、フラットフレームが含まれている場合、それらは登録およびスタッキング処理で使用され（マスターが作成され）ます。
出力ファイルは、リストの最初のライトフレームのフォルダにある Autosave.tif という名前の32ビット有理数TIFFファイルになります。

DeepSkyStackerCL.exe /r /SR c:\MyDSSLists\MyList.txt
ファイルリスト c:\MyDSSLists\MyList.txt 内のすべてのライトフレームを（まだ登録されていない場合のみ）登録し、その後スタックします。
ファイルリストにオフセット/バイアス、ダーク、フラットフレームが含まれている場合、それらは登録およびスタッキング処理で使用され（マスターが作成され）ます。
出力ファイルは、リストの最初のライトフレームのフォルダにある Autosave.tif という名前の32ビット有理数TIFFファイルになります。
さらに、スタック可能な各ライトフレームに対して、ライトフレーム名に .reg.tif または .reg.fts の拡張子が付いたTIFFまたはFITSファイルが作成され、登録およびキャリブレーションされたライトフレームが格納されます。

DeepSkyStackerCL.exe /S /OF16i /OC2 /O:c:\MyDSSResults\Result.tif c:\MyDSSLists\MyList.txt
ファイルリスト c:\MyDSSLists\MyList.txt 内のすべてのライトフレームをスタックします。
ファイルリストにオフセット/バイアス、ダーク、フラットフレームが含まれている場合、それらはスタッキング処理で使用され（マスターが作成され）ます。
出力ファイルは、c:\MyDSSResults\Result.tif という名前の16ビット整数ZIP（DEFLATE）圧縮TIFFファイルになります。

ファイルリストフォーマット
ファイルリストは、次のような構造を持つテキストファイルです。

1行目には DSS file list が含まれている必要があります。
2行目には  CHECKED<TAB>TYPE<TAB>FILE が含まれている必要があります。
ここで、<TAB>はタブ文字（ASCII 09）です。

他の行は、ファイルエントリを説明する場合、次の形式にする必要があります。
[checked]<TAB>[type]<TAB>[full path name]

ここで、[checked] は、フレームがチェックされている場合は 1、チェックされていない場合は 0 です。
（チェックされていないフレームはDeepSkyStackerCLによって無視されます）
ここで、[type] は、ライトフレームの場合は light（またはリファレンスライトフレームの場合は reflight）、ダークフレームの場合は dark または darkflat、フラットフレームの場合は flat、オフセット/バイアスフレームの場合は offset です。
ここで、[full path name] はファイルへのフルパス名です。

行がグループ変更を説明している場合は、次の形式にする必要があります。
#GROUPID#<Group Number>
グループ変更を記述する行の後のファイルから、次のグループ変更までのすべてのファイルは、グループ <Group Number> に配置されます。<Group Number> は正の整数値である必要があります。
グループが指定されていない場合、ファイルはメイングループ（ID 0）に配置されます。

[Save File List...] コマンドを使用すると、DeepSkyStacker から非常に簡単にファイルリストを作成できます。
